GET /health
Health check endpoint to verify service status.
Parameter Type Required Description
No parameters required
Example: /health
GET /api/iClassSchedule
Query course schedule for a specific student and date.
Parameter Type Required Description
studentId String Required Student ID number
studentName String Required Student's full name
dateStr String Optional Date in "YYYYMMDD" format (defaults to today)
Example: /api/iClassSchedule?studentId=1234&studentName=John&dateStr=20251102
POST /api/iClassSign
Sign in for a specific course. Only accepts JSON data.
Parameter Type Required Description
studentId String Required Student ID number
user_id String Required User ID from login session
course_id String Required Course ID to sign in for
classBeginTime String Required Course begin time in "YYYY-MM-DD HH:MM:SS" format
classEndTime String Required Course end time in "YYYY-MM-DD HH:MM:SS" format
Request Body: {
  "studentId": "123456",
  "user_id": "user123",
  "course_id": "course456"
  "classBeginTime": "2025-11-01 08:00:00",
  "classEndTime": "2025-11-01 12:15:00"
}
POST /api/proxy
General-purpose proxy for campus network requests. Requires API key authentication. Only accepts JSON data.
Parameter Type Required Description
API_KEY String Required Your API key for authentication
target_url String Required URL to proxy the request to
target_method String Optional HTTP method (default: GET)
target_headers Object Optional HTTP headers to include
target_params Object Optional URL parameters
target_data Object Optional Request body data
target_json_data Object Optional JSON data for request body
Request Body: {
  "API_KEY": "your_api_key_here",
  "target_url": "https://example.com/api/data",
  "target_method": "POST",
  "target_headers": {"Content-Type": "application/json"},
  "target_json_data": {"key": "value"}
}
